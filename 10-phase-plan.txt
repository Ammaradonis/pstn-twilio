Critical Twilio voice architecture:

Inbound PSTN call:
1. Someone calls a Twilio number owned by this app.
2. Twilio sends a signed webhook to:
   POST /webhooks/twilio/voice/inbound
3. Backend validates the Twilio signature.
4. Backend looks up the Twilio number in the database.
5. Backend creates/updates a call log row.
6. Backend emits a WebSocket event to the owner’s active browser sessions.
7. Backend returns TwiML:
   <Response>
     <Dial>
       <Client>{computed_client_identity_for_number_or_user}</Client>
     </Dial>
   </Response>
8. Browser rings through Twilio Voice JS SDK.
9. User can accept/reject the call in the dedicated number call page.
10. Backend updates call state from status callbacks.

Outbound PSTN call:
1. User opens the dialer page for a selected Twilio number.
2. Frontend requests a short-lived Twilio Voice access token from:
   POST /api/voice/token
3. Frontend initializes Twilio.Device with the token.
4. User enters destination PSTN number.
5. Frontend starts a Twilio Voice SDK call with params:
   - selectedNumberId
   - destinationNumber
   - optional callContextId
6. Twilio requests TwiML from the TwiML App Voice URL:
   POST /webhooks/twilio/voice/outbound
7. Backend validates Twilio signature.
8. Backend validates that the requesting identity is allowed to use the selected Twilio number.
9. Backend returns TwiML:
   <Response>
     <Dial callerId="{selectedTwilioNumberE164}">
       <Number>{destinationNumberE164}</Number>
     </Dial>
   </Response>
10. Backend records call log and updates status via callbacks.

SMS inbound:
1. Twilio receives SMS on one of the provisioned numbers.
2. Twilio sends signed webhook:
   POST /webhooks/twilio/messaging/inbound
3. Backend validates signature.
4. Backend stores the message in PostgreSQL.
5. Backend emits real-time WebSocket event to the relevant number inbox page.
6. Backend returns valid TwiML or 200 response as required by Twilio.

SMS outbound:
1. User opens a number-specific inbox.
2. User enters destination number and body.
3. Frontend calls:
   POST /api/numbers/:numberId/messages
4. Backend validates user authorization, rate limits, validates E.164 number, stores pending message, sends through Twilio Messaging API, updates message status, and emits updates.
